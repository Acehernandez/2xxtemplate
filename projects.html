ASL Letter Translator.

Bilingual dictionaries are usually a two-way street; for example, you could look up a word up in English from another language. Sign-language dictionaries, however, translate only from written words to gestures. This can be hugely frustrating, particularly for parents of deaf children who want to understand gestures, or deaf people who want to interact online using their primary language. Recently, innovations have been made to help this reverse translation, but they require hundreds of dollars worth of equipment. How can we assist ASL to English translation using only equipment found normally?

For the hearing-impaired and their families, learning ASL can be a Catch-22, since there are ASL dictionaries in print, but because the language lacks a written form the signs are often organized according to their nearest English translation. According to Carol Neidle, “You can only look up a sign in the dictionary if you already know what it means.” This is a problem for people regularly using ASL since they cannot fully utilize the English language. Although our project will not directly solve this problem, it will give others the resources they need to tackle the problem by allowing translation to happen at home (using normal equipment.)

Transfer Learning to Recognize Letters via the Webcam

Instructions:

  1. Open up the demo
  2. Wait for the mobilenet model to load; if it doesn't within a minute or so, make sure you're using chrome and have a webcam.
  3. Add examples of letters by clicking and holding on the buttons that say "Add Sample". An image of your most recent sample should show on the button.
  4. Once you have enough examples (100 of each should do), click train model.
  5. Wait for the model to finish training; then press start! The program should highlight the letter it thinks you're signing.
View the Project Source Code

About the demo:

This example shows you how to predict letters from a webcam using transfer learning.
This example is deploying a pretrained Mobilenet model and training another model using an internal mobilenet activation to predict different letters from the webcam defined by the user.
Thank you Tensorflow.js for your flexible and intuitive APIs.
